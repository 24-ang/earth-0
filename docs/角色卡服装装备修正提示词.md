# earth-0 角色服装与装备修正专用提示词

> 角色已经存在——只需要补服装描述和装备细节。
> 把下面的提示词 + 角色图片发给 Gemini（需识图），输出只含 outfits 和 equipment 的补丁 JSON。
> 贴回来让 Claude 帮你合并进 characters.json。

---

## 提示词正文（复制以下全部内容）

你是一个 earth-0 游戏的服装细节补全器。我会给你几张已有角色卡 + 对应的角色图片。每个角色**只输出 outfits 和 equipment 的补丁 JSON**——不要写完整角色卡。

**服装描述有两层——不要混淆：**

| 层级 | 位置 | 谁看 | 粒度 | 例子 |
|------|------|------|------|------|
| `outfits.work.desc` | 每套 outfit 下 | LLM（查角色时） | 整套搭配感 | "修身西装敞开，白衬衫被巨乳撑得纽扣濒临崩开…" |
| `equipment.top.flavor` | 每件装备下 | 玩家（TUI状态面板） | 单件质感 | "浅灰西装，合身但肩部微磨。左胸口袋插红笔。" |

**关键规则：**
- `desc` = 80-150字，面料/版型/穿法/饰品/整体印象。引擎渲染自动跳过，不费tk。
- `flavor` = 20-60字，这件单品的细节、磨损、质感。**状态面板直接显示**。脱了那件就看不到。
- `hair` = 每套 outfit 下加 `hair` 字段，描述这套搭配对应的发型（引擎同样自动跳过，不渲染为装备）。不同 outfit 的发型往往不同：校服披散+蝴蝶结 / 泳装高马尾 / 睡衣松散麻花辫。
- outfit 的每个 **slot 值必须是短名（3-10字）**——长描述一律放进 `desc`
- 每套 outfit 写 3 套（work/school + casual + sleep）
- equipment 每个槽位都要写，不要漏

---

### 🚨 上次犯的错

1. **slot 值写成长文**：`"top": "修身剪裁的浅灰色职业西装外套，通常敞开衣襟……"` 这是 desc！slot 应该是 `"浅灰色西装外套"`。
2. **equipment 全裸——只有 name 没有 flavor**：状态面板只能看到 `[上衣] 浅灰色西装外套`，后面什么都没有。
3. **漏了 sleep 的 desc**：睡觉穿的也有细节（面料/款式/穿了多少年）

### 🎯 提交前自检

- [ ] 每套 outfit 都有 `desc` 和 `hair`（3套 = 3个desc + 3个hair）
- [ ] outfit 的每个 slot 值 ≤ 10 字——长描述没有塞进 slot
- [ ] equipment 每件都带 `flavor`——不光秃秃的名字
- [ ] `hair` 跟 outfit 走——不同 outfit 的发型不同（校服披散/运动高马尾/睡衣麻花辫）
- [ ] 每人一个完整补丁 JSON，name 跟已有角色卡一致
