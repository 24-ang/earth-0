## 三、本轮新实现：Step 6 日程重构 + 通勤偶遇

### 6A：假期多日日程覆盖

**问题**：日历只驱动叙事不驱动日程。暑假/寒假/黄金周——LLM 知道放假，`updateNPCSchedules` 不知道。NPC 照样每天去学校。`org_effects` 只支持单日活动（体育祭→操场），不支持多日覆盖。

**方案**：`CalendarEntry` 加 `schedule_override` + `duration_days`。引擎在模板决议后用覆盖值替换 targetRoom。

**文件变更**：

| 文件 | 改动 | 关键函数/字段 |
|------|------|-------------|
| `engine/types.ts` | `CalendarEntry` + `schedule_override?: Record<string,string>` + `duration_days?: number` | — |
| `engine/state.ts` | 新增 `getActiveScheduleOverrides()` 扫描日期窗口→合并覆盖 | `getActiveScheduleOverrides()` |
| 同上 | `updateNPCSchedules` 内 `!targetRoom` 模板决议后插入覆盖检查 | 约在 effectiveGroup 决议后 |
| `engine/timeline.ts` | `loadCalendar` 从 `function` 改为 `export function` | — |
| `worldpacks/oregairu/calendar.json` | 23→28 条：+暑假42天 +寒假14天 +春假14天 +黄金周5天 +七夕 +花火 +情人节 +白色情人节（含 `schedule_override`） | — |
| `worldpacks/oregairu/schedule_templates.json` | 所有组 `weekday_evening` 从纯 `"自宅"` 改为带选项（自宅/商店街/千葉駅前/図書館/塾/居酒屋...） | — |

### 6B：通勤偶遇叙事注入

**问题**：旅行是瞬间传送——`travel("学校")` → `setPlayerLocation(学校)` → 到了。没有"在路上"的叙事空间。电车上偶遇、放学路口同行——这些恋爱/日常系核心体验缺失。

**方案**：**纯叙事注入。不建新房间、不移动 NPC、不拦截旅行流程。** 旅行正常到达后，引擎检测同方向 NPC → 写入 `_lastCommuteEncounter` → Phase 3 prompt 追加途中偶遇倒叙段。

**文件变更**：

| 文件 | 改动 | 关键函数 |
|------|------|---------|
| `engine/commute.ts` | **新建**：`detectCommuteEncounter()` — 概率检测 + 同方向 NPC 选人 + 叙事文本生成 | `detectCommuteEncounter()` |
| 同上 | `getCommuteDirection()` — 京葉線/総武線分组 | `getCommuteDirection()` |
| 同上 | `getNPCHomeArea()` — 按 `default_location_by_age` 查 NPC 住址 | `getNPCHomeArea()` |
| `engine/types.ts` | `_lastCommuteEncounter?: string` | — |
| `tools/lookup/travel.ts` | 同城到达后 `await detectCommuteEncounter()` → 写入 `_lastCommuteEncounter` | 约 setPlayerLocation 后 |
| `tools/lookup/complete_travel.ts` | 同上 | 约 moveTo 后 |
| `engine/phase3-render.ts` | `_lastCommuteEncounter` → `parts.push()` + `delete` | — |

**详细规则**：

| 维度 | 规则 |
|------|------|
| 基础概率 | 30% |
| 载具修正 | car ×0.1（几乎不偶遇）、bicycle/motorcycle ×0.6、步行 ×1.0 |
| 大势修正 | tension≥3 +15%、stability<0 +10% |
| 时长修正 | duration>15min +15% |

| 好感度优先 | 按 affection 排序，top 3 随机 1-2 人 |
| 路线加成 | 同方向（京葉線/総武線）排序分数 ×1.5 |

### 6D：星期模板区分（本轮最后补充）

**问题**：引擎只有 weekday/weekend 两档，月曜=火曜=水曜=木曜=金曜。水曜短缩早放学、金曜社交高峰——引擎完全不知道。

**方案**：`slotMap` 生成星期前缀 key：`月_afternoon`、`水_afternoon`、`金_evening` 等。模板查表优先 `dayKey`，不存在则回退 `timeKey`。

**引擎改动**（`engine/state.ts` `updateNPCSchedules`）：

```typescript
// 之前
const slotMap = { "morning": "weekday_morning", ... };
const timeKey = isWeekend ? "weekend" : slotMap[time_of_day];

// 之后
const slotMap = { "morning": "morning", ... };
const timeKey = isWeekend ? "weekend" : `weekday_${slot}`;
const dayKey = isWeekend ? "weekend" : `${day_of_week}_${slot}`;

// 查表: dayKey 优先, timeKey 回退
let routeStr = tpl?.[dayKey] || tpl?.[timeKey];
// weather_overrides/seasonal_overrides 同理
```

**数据改动**（`schedule_templates.json`）：

| 组 | 新增 key | 值 |
|----|---------|-----|
| 高校生/总武高学生 | `水_afternoon` | 商店街/千葉駅前/自宅/図書館/ゲームセンター |
| 同上 | `金_evening` | カラオケ/千葉駅前/ファミレス/ゲームセンター/商店街 |
| 运动部员 | `水_afternoon` | 操场/体育馆/商店街 |
| 同上 | `金_evening` | ファミレス/千葉駅前/自宅/カラオケ |
| 社团部员 | `水_afternoon` | 侍奉部/社团楼1F走廊/商店街 |
| 同上 | `金_evening` | カラオケ/千葉駅前/ファミレス/自宅 |
| 总武高教师 | `金_evening` | 居酒屋/ラーメン屋/千葉駅前/自宅 |

**向后兼容**：不写星期 key 的模板组和时段自动回退 `weekday_*`，行为不变。
