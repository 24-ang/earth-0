# earth-0 开局全流程 + TUI 验证提示词

> 把这个粘贴给 GM LLM，它会自己跑完全流程。你只需要看着。

---

## 第一步：删旧档

```bash
rm -rf state/
```

## 第二步：初始化游戏骨架

```
init_game: 玩家姓名=秋月孝三, 性别=男, 年龄=35
```

## 第三步：应用身份模板

```
init_profile: 模板ID=千叶市上班族
```

## 第四步：建住宅（独栋+入住）

```
instantiate_residence: 模板=独栋_2F_4人家庭, 名称=秋月家, movePlayerIn=true, playerRoom=主卧
```

## 第五步：验证基本状态

依次输入以下命令，确认返回正常：

1. `/look` — 应该显示秋月家主卧的空间描述和网格
2. `/status` — 应该显示秋月孝三的完整状态（装备、身体、关系）
3. `/bag` — 应该显示背包物品（手机、书包），每个物品有名称和重量
4. `/room` — 应该显示当前房间的家具和空间

## 第六步：验证关键修复

逐项确认：

1. **gridPos 不 null**：`/move 北` 应该返回移动结果，不是"gridPos null"报错
2. **导航可用**：`/go 秋月家客厅` 应该成功设置 pendingTravel，不报"未找到"
3. **物品有价格**：`/bag` 里的物品应该能正常显示（不需要显示价格数字，但系统能读取）
4. **body 槽位**：`spawn_item 白衣` 然后 `/bag` 过滤已装备，白衣应该能穿上
5. **关系无自引用**：`/status` 的关系表里不应该有秋月孝三对自己的好感度
6. **住宅子房间**：`/go 秋月家浴室` 应该能找到（不报"未找到"）
7. **千叶_住宅区可通行**：`/go 校门` 或 `/go 商店街` 应该能找到目的地

## 第七步：手动存档 + 重启验证

1. `/save` — 手动存档
2. 退出游戏
3. 重新进入游戏（会自动 loadState）
4. `/look` — 确认 gridPos 还在、位置正确、住宅还在
5. `/status` — 确认状态没有回退

## 如果某步失败

告诉我具体哪一步、返回了什么错误信息。不要再自己改代码——问题收集完我来统一修。

---

## 已确认不修的问题（不是 bug）

- **爱莉好感10但 stage="陌生"**：关系阶段由叙事推进，引擎不强制覆盖
- **35岁 isVirgin: true**：引擎不替 LLM 做性经历判断（PHILOSOPHY §1.2）
- **手机联系人**：init_profile 末尾自动从已建立的关系同步联系人（无好感门槛）。手机里只会有你真正认识的人，不会乱塞跨作品角色
- **职业身份**：`set_flags(is_teacher=true)` 后引擎会自动授予"教师"称号（显示在 [称号] 段），不需要额外的身份设置
