# earth-0 开局全流程 + TUI 验证提示词

> 把这份文档粘贴给 GM LLM，它会按顺序调用工具跑完全流程。你只需要看着，有报错贴给我。

---

## 第一步：开始新游戏

直接在聊天框里说：

```
新游戏：我叫秋月孝三，男，35岁
```

GM 会自动调用 `init_game`。等待初始化完成后继续说：

```
我是千叶市上班族
```

GM 会自动调用 `init_profile`。然后说：

```
给我一栋独栋房子，叫秋月家，我自己住进去，住主卧
```

GM 会自动调用 `instantiate_residence`。

---

## 第二步：验证基本状态

依次在聊天框输入以下命令：

1. `/look` — 应该显示秋月家主卧的空间描述和网格
2. `/status` — 应该显示秋月孝三的完整状态（装备、身体、关系）。**装备每件应有 flavor 描述**
3. `/bag` — 应该显示背包物品（手机、书包、公文包），**每件有名称+描述+重量+体积**
4. `/room` — 应该显示当前房间的家具和可交互动作

---

## 第三步：功能逐项验证

在聊天框里依次输入以下内容，**每项完成后看返回结果是否符合描述**。不符合就记下来告诉我。

### 空间系统

- `/move 北` — 返回移动结果，不是报错
- `/go 秋月家客厅` — 成功设置行程，不报"未找到"
- `/go 秋月家浴室` — 能找到住宅子房间
- `/go 校门` — 能找到外部目的地
- 对着客厅的沙发说 `我要坐沙发上休息` — GM 应调用 interact_furniture，返回正确交互
- `lookup_furniture search=冰箱` — 应返回冰箱定义（container+switchable+cold_storage）

### 角色数据

- `/schedule` — 应显示高校生/总武高学生而非"小学生"
- `lookup_character 平冢静` — schedule_group 应为总武高教师，default_location 为职员室

### 身体暴露

- `/status` — 穿全套装备时不应暴露性征数据
- 说 `我把下身衣服全脱了` — GM 调用 equip_item 脱掉 bottom+inner_bot，之后 `/status` 应显示性征

### 存档

- `/save` 然后 `/look` — 确认位置正确
- 退出重进后 `/look` — 位置和装备还在

### 全局大势

- 随机走动或触发一轮结算后，GM 旁白应能看到 `[全球大势]` 段

### Vicky 政治经济学系统

- 在总武高/侍奉部附近走动 → GM 旁白应出现 `[本地活跃势力]` 段，含 🏠/★/●/○ 标记
- 同 tier 同 sector 的组织之间可能出现 `⚔️ 竞争` 或 `🤝 合作` 标记

### 玩家→组织互动

- `contribute_to_org orgId=soubu_service_club action=donate amount=1000` — 返回捐款成功，org 财力增加
- `contribute_to_org orgId=soubu_service_club action=complete_quest details=帮忙解决了一个委托` — 凝聚力+5，声望+3
- `lookup_org orgId=soubu_service_club` — 返回侍奉部的分级信息（中立看到基本信息，声望高了能看到更多）

### 组织创建

- 说 `我想创建一个新的社团，叫轻音部，在音乐教室活动` — GM 应调用 create_organization

---

## 第四步：自检报错

看控制台。如果出现以下格式的文字，复制完整的行给我：

- `[validate-state][ERROR]` — 严重问题
- `[validate-state][WARN]` — 需要关注

---

## 如果某步失败

告诉我：哪一步、你输入了什么、返回了什么。不要再自己改代码。
