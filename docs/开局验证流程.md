## 第五步：验证基本状态

依次输入以下命令，确认返回正常：

1. `/look` — 应该显示秋月家主卧的空间描述和网格
2. `/status` — 应该显示秋月孝三的完整状态（装备、身体、关系）。**装备每件应有 flavor 描述**
3. `/bag` — 应该显示背包物品（手机、书包、公文包），**每件有名称+描述+重量+体积**
4. `/room` — 应该显示当前房间的家具和可交互动作

## 第六步：验证关键修复（2026-07-05 累计）

逐项确认：

### 空间系统

1. **gridPos 不 null**：`/move 北` 应该返回移动结果，不是"gridPos null"报错
2. **导航可用**：`/go 秋月家客厅` 应该成功设置 pendingTravel，不报"未找到"
3. **住宅子房间**：`/go 秋月家浴室` 应该能找到（不报"未找到"）
4. **千叶_住宅区可通行**：`/go 校门` 或 `/go 商店街` 应该能找到目的地

11. **spawn_item 超重拦截**：尝试 `spawn_item 丰田卡罗拉`（~1300kg）应被拒绝

### NPC 与路人

12. **路人时段/地点过滤**：在学校教室时，不应出现主妇/流浪汉/外卖员
13. **群演感知注入**：GM 旁白和 NPC Agent prompt 中应能看到 `[在场路人]` 段
14. **NPC 环境感知**：spawn NPC Agent 时应看到天气/季节/时段/房间家具信息
15. **NPC 身体暴露**：脱掉 NPC 的 bottom+inner_bot 后，其他角色应能看到性征数据

### 角色数据

### 存档与恢复

19. `/save` → 退出 → 重进 → `/status` 确认状态没有回退
20. `/save` → 退出 → 重进 → 确认 body 暴露状态正确恢复

### 身体暴露证明

21. **穿着整齐**：秋月孝三穿全套装备时，NPC 不应看到性征数据

### 幕间/切镜系统（2026-07-05 新增）

25. **replay_pov 同场复述**：spawn_npc_agent 让 NPC 说一句有份量的台词 → 同回合内调 `replay_pov(该NPC名)` → 下一回合出现【切镜 · NPC名】，内容为该台词的慢镜头回放

## 已确认不修的问题（不是 bug）

- **爱莉好感10但 stage="陌生"**：关系阶段由叙事推进，引擎不强制覆盖
- **35岁 isVirgin: true**：引擎不替 LLM 做性经历判断（PHILOSOPHY §1.2）
- **手机联系人**：init_profile 末尾自动从已建立的关系同步联系人（无好感门槛）
- **职业身份**：`set_flags(is_teacher=true)` 后引擎自动授予"教师"称号
- **NPC 移动是瞬移**：日程切换时 NPC 瞬间换位（非实时寻路），但门关了/玩家堵门/家具堵出口 → NPC 被拦住

## 2026-07-05 提示词架构重整 + 幕间系统补全

### Phase 1 prompt 瘦身
- 工具列表从手写 param 细节 → 按功能分类（🚶移动/👥群演/🏠场景/💬关系/🛒经济/⚔️冲突/🎒物品/🚗载具/📋管理/🆕新游戏/🎬镜头）
- `buildStartupPrompt` 从 `buildClassificationPrompt` 分离为独立函数

### 工具层描述收紧
- `spawn_item`: effects.type/value 补齐枚举和说明，flavor 加示范 + "缺少则物品在叙事中无存在感"
- `create_character`: schedule_group 从 5 假枚举 → 14 真枚举；appearance_brief/outfits 补叙事重要性说明
- `init_profile`: 描述说明模板给骨架、引擎从 items.json 补 effects/flavor 的机制

### 引擎层兜底
- `defaultUnderwear()` 引擎创建的内衣自带 flavor（不再依赖 items.json 查找）
- `normalizeItem()` 三级兜底：模板 flavor > items.json > "一件普通的{name}"——永不空
- `spawn_item` execute：LLM 不传 flavor 时引擎补 "一件普通的{name}。来源: xxx"

### 幕间/切镜系统
- **replay_pov** 工具激活（路径 5：同场复述）——标记 NPC 发言 → 下回合慢镜头重放
- **声望切镜正负基调(C)**：tone/topic 按声望等级(0~5)×方向(升/降) 分化 10 种基调
- **声望切镜选人 npcBelongsToOrg**：硬编码 name list → 通用组织匹配（与日历/org_effects 同一入口）
- **LLM 动态任务+幕间打通**：DynamicEvent 加 intermission 槽位；advanceQuest 三个断点修复（查 dynamicEvents / 无 beat 直接完成 / 完成时消费 intermission）
- **幕间 prompt 数据补厚**：有角色卡 NPC → 注入性格/外貌/穿着/身体/关系/记忆/印象；无角色卡 → 只写名字+trigger 不崩；环境数据始终注入
- **合同 + Phase1 prompt 指引**：gm-intermission-contract.md 加"适用时机"段；Phase1 prompt 加 🎬镜头工具+五条使用判断+世界事件开场指引
