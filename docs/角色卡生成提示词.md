# earth-0 角色卡生成提示词

> 把下面的完整提示词 + 角色图片发给 Claude/Gemini（需识图功能），它会输出 JSON。
> 一次可发多张图，每张生成一张卡。拿到了贴回来，我帮你塞进 `worldpacks/oregairu/characters/{角色名}.json`（一人一文件）。

---

## 提示词正文（复制以下全部内容）

你是一个 earth-0 游戏的角色数据生成器。earth-0 是一个千叶市为舞台的文字 RPG 沙盒引擎。

我会给你几张角色图片。请你根据图片中角色的外貌、气质、可能的年龄和身份，**每人输出一份**以下格式的 JSON 对象。文件名用角色名.json。

**服装描述有两层——不要混淆：**

| 层级 | 位置 | 读的人 | 粒度 | 例子 |
|------|------|--------|------|------|
| `outfits.work.desc` | outfits 每套下一个 | LLM（lookup_character 查角色时） | 整套搭配感 | "修身西装敞开，白衬衫被巨乳撑得纽扣濒临崩开…" |
| `equipment.top.flavor` | equipment 每件下一个 | 玩家（TUI 状态面板直接显示） | 单件质感 | "浅灰西装外套，合身但肩部微磨。左胸口袋插红笔。" |

- **`desc`**：引擎渲染时自动跳过，不浪费 token。只在 LLM 调用 `lookup_character` 时输出。
- **`flavor`**：状态面板显示 `[上衣] 浅灰色职业西装外套 — 浅灰西装外套，合身但肩部微磨…`。每件可被单独脱下/偷走/损毁——flavor 跟装备走，脱了那件就看不到了。不像 `desc` 是全套一体的。

---

### 🚨 上次别人犯的错（你不要再犯）

1. **`default_location` 是「家」不是「上班的地方」**。藤崎穗成美上次被写成了 `"总武高_游泳池"`——错。她的家应该在千叶住宅区（海浜幕張/稲毛海岸等），上班才去游泳池。`default_location` 永远填住宅地址。
2. **`social_class` 只有五个合法值**：`"大资产阶级"` / `"小资产阶级"` / `"无产阶级"` / `"知识分子"` / `"流氓无产阶级"`。上次写了 `"中产阶级"`——非法，自创的，引擎不认识。教师统一填 `"知识分子"`。
3. **以下字段上次被整段省略了**：`body_by_age` / `schedule_group_by_age` / `default_location_by_age` / `personality_stages` / `drives_by_age` / `appearance_by_age` / `private_facts`。**每一个都必须写**，哪怕只写成年年龄一个节点。
4. **不要用旧格式 `schedule` 数组**。这个系统用 `schedule_group` + `schedule_group_by_age`——不是手写每个小时的行程表。
5. **`body_shape.chest` 合法值**：`"半球"` / `"水滴"` / `"圆盘"` / `"纺锤"` / `"吊钟"` / `"扁平"`。不要写 `"巨乳"`——那是形容词不是形状。
6. **`body_by_age` 必须至少 4 个节点**：6岁 / 12岁 / 15岁 / 成年年龄。每个节点都要写完整（身高/体重/build/cup(女)/三围(女)/leg_type/skin）。每个年龄的三围和 cup 必须是递进的——6岁不可能有 I cup。
7. **outfit slot 值必须是短名（3-10字），不要把长描述写进 slot**。上次有人把 `top` 写成了 80 字的面料版型穿法大全——那是 `desc` 的事，不是 `top` 的事。slot=短名（引擎渲染用），desc=长文（LLM查细节用）。
8. **equipment 每件必须带 `flavor`**。上次所有装备全裸——只有 `name` 没有 `flavor`，玩家在状态面板看装备只能看到一个光秃秃的名字。flavor 跟装备走——脱了那件就看不到了。

### 填写规则

**必须精确的字段：**

| 字段 | 规则 |
|------|------|
| `name` | 不要带空格和特殊符号 |
| `nationality` | 国籍/出身。从现实千叶JR沿线真实社区中选择。韩裔→海浜幕張/千葉みなと有韩国人社区。归国子女也常见。如"韩裔日本人（首尔出生，小学时搬来千叶）""千叶本地""中韩混血" |
| `occupation` | 具体职业/身份，比 schedule_group 更具体。如"崔氏道场跆拳道师范""总武高2年J班学生·拳击部""千葉みなと仓库区夜班打工""家庭主妇""无职" |
| `base_age` | 教师通常 25-55 |
| `default_location` | **必须从千叶真实JR沿线地址中选一个**：海浜幕張 / 稲毛海岸 / 千葉駅前 / 西千葉 / 検見川浜 / 蘇我 / 千葉みなと / 本千葉 / 稲毛 |
| `schedule_group` | 教师 = "总武高教师"，上班族 = "上班族"，自由业 = "自由人" |
| `grade` | 几年级。高一=1 高二=2 高三=3。如果是总武高学生必须填。教师不用填。 |
| `homeroom` | 哪个班(A~J)。雪乃是J班、八幡是F班、一色是C班。教师不用填。按年龄分班: 16岁→2年、15岁→1年、17岁→3年 |
| `schedule_group_by_age` | 必须覆盖从 6 到 base_age 的所有关键年龄段。例如 28 岁：{"6":"小学生","12":"中学生","15":"高校生","18":"大学生","22":"总武高教师","28":"总武高教师"} |
| `default_location_by_age` | 至少覆盖 6岁/12岁/15岁/成年 四个节点 |
| `social_class` | 教师 = "知识分子"（公立学校教师），私立/精英教育工作者 = "小资产阶级" |
| `personal_axes` | 经济立场 -5(左翼/平等) 到 +5(右翼/市场)；政治立场 -5(进步/自由) 到 +5(保守/传统) |
| `body_by_age` | 必须含 6/12/15/成年 四个节点。女教师需含 cup 和三围按年龄的变化 |
| `attributes` | 六维1-20。成年人默认不低于8。智力/魅力通常较高，力量/敏捷较低 |
| `HP` | max = 体质×2（成年教师通常 12-20） |
| `outfits` | 教师用 "work" 键替代 "school"。写实风——不需要校徽和水手服。如：灰色西装外套/白衬衫/深灰西裤/皮鞋 |

**可以简化的字段：**

| 字段 | 简化方式 |
|------|---------|
| `outfits` | 写今天的日常穿着即可。教师通常：work(上班装)、casual(便服)、sleep(睡衣) 三套。**每套必加 `desc` 字段**：短名供引擎渲染，desc供LLM调用lookup时查细节（80-150字的面料/版型/穿法/饰品/整体印象描述）。引擎渲染时自动跳过desc——不费tk。 |
| `skills` | 选 3-5 项与角色相关的 |
| `public_facts` | 1-2 条全校师生知道的 |
| `private_facts` | 1-2 条只有同事或密友知道的 |
| `drives_by_age` | 教师通常不需要写少年时期——只写成年的就行 |
| `equipment` | 穿在身上的日常装备。**每件必加 `flavor`**：这件单品的面料、版型、穿法、磨损痕迹、特别之处（20-60字）。flavor在TUI状态面板直接显示——玩家查看每件衣服都能看到细节。outfit的`desc`是整套搭配感，equipment的`flavor`是单件质感——两层互补。 |
| `inventory` | 手机+钱包+个人物品就够了 |

**男性教师可以省略：**
- `cup`（设为 null 或不填）
- `measurements`（三围不需要）
