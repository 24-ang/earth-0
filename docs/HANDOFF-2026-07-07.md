## 三、下一步任务（按优先级）

### 🔴 优先级1：`contribute_to_org` 工具

**新建** `tools/action/contribute_to_org.ts`。这是**玩家→org 互动的缺失环节**。现在玩家帮侍奉部做事只有声望变化，org本身的wealth/cohesion完全不动。

参数设计：
```typescript
{
  orgId: string,                                    // 目标组织ID
  action: "donate" | "complete_quest" | "betray" | "recruit_member",  // 贡献类型
  amount: number,                                   // 捐赠金额（仅donate）
  details: string,                                  // 描述（任务名/情报内容）
  targetNpc?: string                                // 拉拢/背叛的目标NPC
}
```

效果：
- donate → org.wealth + (amount/100) + 玩家声望+N
- complete_quest → org.cohesion +5 + org.influence +2 + 玩家声望+3
- betray → org.cohesion -10 + org.public_legitimacy -5 + 敌对org声望+N
- recruit_member → org.members 数组 push + org.cohesion +3

注册到 `tools/registry.ts` 的 trackedTools，加进 Phase 1 白名单和 prompt。

### 🔴 优先级2：NPC 反应式日程越权

设计文档在 `C:\Users\Xiang\.gemini\antigravity\brain\433f4731-90be-4cbb-97b3-707011bf8e50\npc_reaction_schedule_design.md`。

核心：玩家恶意行为触发 NPC 的日程覆盖，引擎已有的 `pendingOverride` 是基础设施。

实现位置：在 `engine/state.ts` 或新建 `engine/npc-reactions.ts`，在 `updateNPCSchedules` 前扫描玩家的恶意行为记录，自动写 `pendingOverride`。

四种模式：avoid（绕开玩家）、tail（跟踪玩家3-5回合）、confront（直接对质）、setup（找靠山帮忙）。

### 🟡 优先级3：org scale 运行时升降级

现有 `scale` 是静态的。需要加 `effectiveScale` 或让 `scale` 可写。

升降条件：
- 升级：`influence >= 60` 且 `cohesion >= 70` 且持续 >= 5 回合 → scale 升一级
- 降级：`cohesion < 20` 且持续 >= 3 回合 → scale 降一级
- 解散：`wealth < 5` 且 `cohesion < 5` 且 `public_legitimacy < 10` → 标记为 archived

实现位置：在 `evaluateOrgGoals` 或 `applyWorldStateToOrgs` 里加阶段判定。

### 🟢 优先级4：完整 Org 生命周期

六个阶段的状态转换逻辑，在 `evaluateOrgGoals` 里根据 wealth/influence/cohesion 的阈值判断当前阶段，注入不同的叙事信号和引擎行为。
