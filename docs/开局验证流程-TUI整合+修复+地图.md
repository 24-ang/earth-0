# TUI 整合 + 交互修复 + 新地图 验证提示词

> 用法：关游戏重开，逐条粘给 GM / 敲斜杠命令。看输出 + 控制台。
> 改了什么：13 个面板并成 /info、/economy、/status 三个入口；修一批"点了没反应/开不了"的交互 bug；新增栄町、稲毛海滩两处地点。
> 最后更新：2026-07-12

---

## A. 整合后的新命令

### 1. /info 信息中枢（原 calendar/quest/alerts/schedule/weather/memory 六合一）
```
/info
```
✅ 应弹出菜单，含 6 项：日历与事件 / 任务与剧情 / 警报状态 / NPC 日程 / 天气 / NPC 记忆。逐个点开都能显示内容，不空、不崩。
❌ 若菜单空、或点某项没反应/报错 → 贴哪一项 + 控制台。

### 2. /economy 经济中枢（原 shop/gamble/housing 三合一）
```
/economy
```
✅ 应弹出菜单：商店与打工 / 博弈与黑市 / 房产与储物。逐个点开有内容。

### 3. /go skip 跳过剧情直达（原 /goskip）
```
/go skip
```
✅ 打开导航菜单，选个远地点 → **直接到达**，不进入"旅行中等 complete_travel"的排队状态。
对比普通 `/go` 选远地点 → 应设 pendingTravel、提示描述路上见闻。

### 4. /status 收编发育/战力/队伍
```
/status
```
✅ 面板里除了装备/背包，应多出「📈 发育」「⚔️ 战力」「👥 队伍」三个入口，点开分别显示对应子面板。
✅ 卸下/装备一件装备后，列表应立即刷新（槽位变空/变满），再点不会重复。

### 5. 旧命令已移除
```
/calendar
```
```
/shop
```
```
/goskip
```
✅ 这些应**不再弹出面板**（已并入上面的中枢；作为普通文本发给 GM 也可以）。
❌ 若还能弹出旧的独立面板 → 说明没删干净，贴出来。

## B. 修好的交互 bug

### 6. 家具抽屉能打开 + 取放（原来打不开）
先回到玩家房间（有书桌），观察场景：
```
回我自己的房间
```
```
/room
```
✅ 家具列表里应有「书桌」，它那行应显示容器预览（如 `[桌面:… | 抽屉:…]`）。
走到书桌旁边再打开它 → 应能看到「桌面」「抽屉」子容器，能取出/放入物品。
❌ 若书桌打开后没有任何容器可选、或预览显示 `furniture-家_玩家房间-…` 这种丑长内部 id → 贴出来。
（注：得**走到家具旁边**才会出容器，这是设定不是 bug。）

### 7. 货架能显示（原来永远"附近没商店"）
到有自动贩卖机/便利店的地方（如街道、便利店），然后：
```
/economy
```
选「商店与打工」。
✅ 应列出货架商品（如自动贩卖机的 MAX COFFEE/矿泉水…），不再永远"附近未匹配到商店"。

### 8. /achievements 不再自相矛盾
```
/achievements
```
✅ 应显示「成就系统尚未配置」，**不应**同时出现"已解锁 0/0"和"所有成就已全部解锁！"这种矛盾话。

### 9. 菜单灰选项不再误关整个菜单
```
/npc 雪之下雪乃
```
在互动菜单里，对着一个**灰的（条件不足）**选项按回车（如好感不够的"亲吻"）。
✅ 菜单应**保持打开**、什么都不发生（顶多提示条件不足）。
❌ 若按一下整个菜单就关掉了 → 没修好，贴出来。

### 10. 家具"使用/查看/坐下"有反应、"阅读"不崩
```
/room
```
对一个家具点「查看」「坐下」「使用」。
✅ 每个都应有可见反馈（提示文字），不再"点了没反应"（空提示）。
对一个可读家具（书架/书桌）点「阅读」。
✅ 应正常给经验/推进时间，**不崩**、不静默失败。

## C. 新地图

### 11. 去栄町（风俗街）
```
我要去栄町
```
或 `/go` 里找栄町。
✅ 应能到达，显示夜之街的 atmosphere（霓虹/拉客/香水拉面味）。到了后 `/room` 能看到店面入口（夜蝶俱乐部/情人旅馆）。

### 12. 去稲毛海滩
```
我要去稲毛海滩
```
✅ 应能到达，显示海滩/防波堤/海浪的 atmosphere，能原路走回千葉駅前。

---

## ⚠️ 不用报的已知项

- `/reroll` 生成的叙事只在面板里显示、不进正史——已知设计残缺，不用报。
- `/npc` 偷窃、`/room` 移动/家具交互**不推动正文叙事**——已知设计（只有 /npc 的对话/接触类推正文），不用报。
- 家具容器要"走到旁边"才出现——设定，不用报。
- e2e-init 千葉駅前预存 bug——已知。

---

## 失败格式

第几步、你输入了什么、返回了什么、控制台异常行完整复制。
