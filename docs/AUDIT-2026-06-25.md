## 第 0 题：装假狗清单（按严重度排序）

> 定义：声称生效但实际没生效，或描述与实现不符。

### 🔴 D1. 剧情/驱动/人生事件引擎没接到主回合（最严重的装假狗）
- **现象**：`checkTimelineEvents()` / `expireHooks()` / `checkDriveDrivenHooks()` / `tickLifeEvents()` **只在 `tools/helpers.ts:234-238` 的 `advanceTimeMinutes` 里被调用**。
- 而 `advanceTimeMinutes` 只有三个触发点：`/go` 长途旅行、`complete_travel`、`work_job`（helpers.ts:271 / complete_travel.ts:17 / work_job.ts:14）。
- **主回合工具 `commit_turn`（tools/state/commit_turn.ts）和 `settle_scene`（tools/action/settle_scene.ts）都不调用它们**——它们只调 `updateNPCSchedules()`。
- **后果**：玩家在侍奉部坐一下午聊天（走 settle_scene），时间线事件不扫描、钩子不生成也不过期、NPC 驱动钩子不触发、人生事件不 tick。**49 个时间线 JSON（data/timelines + worldpacks）在日常play中基本是死数据。**
- **这正是 `framework-optimization-log.md` 优化③声称已修复的"引擎写了但没接入"问题——它回归了，或从未在主路径生效。**
- **验证**：`grep -rn "checkTimelineEvents" engine/ tools/` → 只有 helpers.ts:234 一处调用点。

### 🔴 D2. 手机时钟读死字段，恒为 undefined
- **`tools/helpers.ts:591-594`** 手机 TUI 顶栏读 `time.year` / `time.month` / `time.date` / `time.day` / `time.hour` / `time.minute` / `time.season`。
- **`engine/time.ts:21` 的 `TimeState` 只有** `game_date` / `day_of_week` / `time_of_day` / `minute_of_day` / `player_age` / `player_stage` / `timeline_origin`。
- 实际存档 `state/session.json` 的 `time` keys 印证：**那 7 个字段从不存在**。
- **后果**：打开手机看到 `undefined年undefined月undefined日 星期undefined undefined:undefined`。tsx 只剥类型不校验，所以静默穿帮。

### 🔴 D3. `recordTurnLog` 时间戳同样读死字段
- **`engine/state.ts:165`**：`timestamp: ${time.year}年${time.month}月${time.day}日 ${time.timeOfDay ?? ""}` —— 这四个字段都不存在（真字段是 `game_date` + `time_of_day`）。
- **后果**：每一条回合台账的时间戳都是 `undefined年undefined月undefined日`。台账是 Layer 2 的核心交付物，时间戳全废。

### 🟡 D4. `record_turn_log` 的 `toolsCalled` 永远是空数组
- **`tools/action/record_turn_log.ts:21`**：`toolsCalled: [], // 引擎自动从本轮工具调用补充`
- 但 `engine/state.ts:161 recordTurnLog` **从不填充它**。注释撒谎。
- **后果**：roadmap Layer 2 设计的"用 toolsCalled 对账幻觉"能力完全不存在。

### 🟡 D5. `render_scene` 的"两段式渲染"是表演
- **`tools/action/render_scene.ts:47` 算出 `logicModel`，然后整个函数再没用过它**。
- `:70-71` 注释直言："不需要 LLM 额外调用逻辑模型，直接用纯代码…拼接 `<directors_note>`"。
- 描述却写"引擎调用渲染模型（Flash）产出"——单模型自调用，根本没有逻辑模型那一段。
- **不算纯假**（确实调了一次叙事模型渲染），但框架叙述夸大。PLAN-v2 把 Layer 4 标成 ✅ 是高估。

### 🟢 D6. 6 个工具改了内存不落盘（crash 窗口丢数据）
`add_calendar_event`(:27) / `reveal_secret` / `toggle_layer1` / `spawn_temp_npc` / `spawn_npc_agent`(:280) / `spawn_npc_agents`(:432) —— 改了 gameState 但自己不调 `saveState()`，靠下一个工具兜底。在会话内有效，硬崩则丢。

**统计**：纯假工具 **0 个**；接线层装假狗 **5 处**（D1-D5，D1/D2/D3 是真 bug）；半成品 **6 个**（D6）。

## 第 1 题：最大架构风险 — 数据源脑裂（Split-Brain Data）

### 事实链
1. `engine/state.ts:4119` 模块加载时执行 `loadActiveWorld()`。
2. `data/.active_world` 是**空文件（0 字节）** → `world=""` falsy → 回退 `"oregairu"` → 加载 **`worldpacks/oregairu/`**（state.ts:4033-4074）。
3. characters / rooms / items / sex_profiles / shops 等 **18 类数据是"替换式"**，运行时只认 worldpacks。
4. **但最近 20 个 git 提交（不知火舞角色卡）全部写进 `data/characters.json` + `data/sex_profiles.json`。**
5. 核验：`data/characters.json` 与 `worldpacks/oregairu/characters.json` **diff 97 行**；不知火舞 sex_profile 在 `data/` 有、在 `worldpacks/` **没有**。

### 后果
**过去几天所有角色卡编辑，运行时根本加载不到。** 玩家进游戏遇到的不知火舞用的是 worldpacks 里的旧版/缺失版。这是"以为在干活，其实在对着影子改"。

### 复杂之处（别一刀切）
- `timelines` / `lore` / `orgs` / `calendar` 是**合并式**（`data/` + `worldpacks/` 都读，见 lore.ts:14+25、timeline.ts:17+275）。
- 所以不能简单删掉 `data/`——部分子系统还真从它读。
- 这种"一半替换一半合并"的混合规则**本身就是风险**：没有单一可信源（single source of truth）。

## 第 3 题：引擎硬约束应统一成什么原则

现状：硬约束散落且摇摆——build_wall 校验背包材料（硬），但经济/物价让 LLM 定（软）；create_room 扣钱（硬），但 D6 一堆不落盘（半软）。`gm-rules.md:225` 的"引擎硬约束"表和实际代码也对不齐。

### 建议统一为一句话原则：

> **引擎守恒，叙事自由（Engine conserves, narration is free）。**
> 引擎只拦截**不可逆**或**有守恒量**的操作；一切可逆、纯风味、可由文字承载的，全部放行给 LLM。

### 落地为可判定的"守恒量清单"

引擎**必须**拦截并记账的，只有这 6 类守恒量：

| 守恒量 | 引擎职责 | 不管什么 |
|--------|---------|---------|
| **钱** funds | 加减必须 ≥0，转移必须双边平账 | 价格高低（LLM 按世界观定） |
| **物** inventory/容器 | 转移守恒（A 减则 B 加）、容积/负重上限 | 物品长什么样、值不值（flavor 归 LLM） |
| **HP / 资源池** | 不可为负、死亡豁免、冷却 | 伤害叙事、技能特效 |
| **时间** | 单调递增、跨天结算 | 这段时间发生了什么 |
| **位置 / 空间** | 网格边界、墙体阻挡、移动耗时 | 路上风景、房间氛围 |
| **可见性** visibility | 秘密分级、未揭示不泄露（lint 兜底） | 怎么揭示、揭示的措辞 |

### 三条铁律（写进 gm-rules.md，替换现有摇摆表述）
1. **凡是守恒量，必须经工具改；工具拒了就是拒了，叙事不得绕过。**（已有，保留）
2. **凡是非守恒量（风味/价格/材料种类/氛围），引擎一律不校验，全权交 LLM。**（明确化，消除摇摆）
3. **每个改守恒量的工具，最后一行必须 `saveState()`。**（机械纪律，根治 D6）

这条原则可判定、可写测试（"这个操作动了守恒量吗？"），能终结边界摇摆。
