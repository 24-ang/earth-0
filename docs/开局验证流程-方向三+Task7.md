# 方向三 + Task#7 验证提示词（去白名单 + 钱拆分 + 默认装备）

> 用法：先关游戏重开，再逐条粘给 GM。看输出 + 控制台。
> 改了什么：买/卖/偷不再硬拒"合不合理"→交 LLM；NPC 钱拆成钱包现金+存款；没填衣服的角色有默认穿着。
> 最后更新：2026-07-09

---

## 1. 买东西不再被"店不卖"挡

```
我在便利店买一台笔记本电脑，出价 80000
```
✅ 应成交（不再"便利店不卖笔记本电脑"）
❌ 若被拒 → 贴返回文本

---

## 2. 卖东西不再被"店不收"挡

先获取一件物品，然后卖掉：
```
给我一双旧运动鞋
```
```
把这双旧运动鞋卖给便利店，卖 500
```
✅ 应成交
❌ 若"不收" → 贴返回文本

---

## 3. 偷钱包——连现金一起，但钱从她身上扣

让一个 NPC 在场，偷她钱包（注意：她钱包里现金是有限的）：
```
由比滨结衣和我在一起
```
```
我偷由比滨结衣的钱包，里面大概有 5000 块
```
✅ 应偷到钱包物件 + 现金（金额不会超过她实际钱包现金）
然后用 `/npc 由比滨结衣` 看她钱包余额——应减少

---

## 4. 偷兰博基尼——引擎拒绝但有叙事

```
我偷由比滨结衣的兰博基尼
```
✅ 应被拒，但拒绝是**叙事形式**（如"这东西太大没法顺手带走"），不是一句冷冰冰的系统提示
❌ 若成功偷到一辆车进背包 → 贴结果

---

## 5. 偷过的东西不能再偷（防重复）

对同一个 NPC 偷两次"钱包"：
```
我再偷一次由比滨结衣的钱包
```
✅ 第二次应被拒："身上已经没有钱包了"

---

## 6. /npc 面板显示钱包现金 + 总身家

```
/npc 秋月孝三
```
✅ 应显示类似 `💰 总身家: ¥1200000 (现金¥5000)`（有钱人钱包只揣 5000，百万在存款里）
对比一个穷人：
```
/npc 鹤见留美
```
✅ 总身家 500，现金几十块

---

## 7. 没填衣服的角色有默认穿着——不裸

```
不知火舞出现在我面前
```
✅ 她应有穿着描述（可能是普通的"女士上衣/裙子"之类），**不应**出现裸体/生殖器描写

---

## 8. 偷她钱包——只看钱包里的现金

```
/npc 不知火舞
```
看她钱包余额，然后：
```
我偷不知火舞的钱包
```
✅ 偷到的现金不超过她钱包里的余额
❌ 若偷到几十万（她可能根本没那么多现金）→ 贴结果

---

## 9. LLM 创建的新角色——有默认衣服，不裸

```
创建一个新角色，叫铃木一郎，男，10岁小学生，原因：测试默认装备
```
```
查一下铃木一郎
```
✅ 应有穿着（如小学生校服/学生鞋），不是空装备

---

## ⚠️ 不用报的已知项

- "钱包是空的"/"这人没什么钱"——如果 NPC 钱包现金确实很少/为0，这是对的（钱拆分后钱包≠全部身家）
- `[_equipmentSource: auto]`——看到这个标记是正常的，表示装备是引擎自动填的
- e2e-init 千葉駅前失败——预存 bug

---

## 失败格式

第几步、你输入了什么、返回了什么。控制台异常行完整复制。
