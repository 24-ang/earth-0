# Phase 2/3 验证提示词（合并前实机）

> 用法：每步引号内容粘给 GM，看输出 + 控制台。有问题按最后一节格式贴回。
> 只验这轮真改过的东西：换装装备同步、儿童开局身材、学龄前日程、空装备不误判全裸。
> 参考模板：`docs/开局验证流程.md`。最后更新：2026-07-09

## A. 换装后装备栏同步（Phase 2a，`ff54646`）

**改了什么**：以前只有雪乃换装时装备栏会变，其余 108 个角色换了衣服但 `/status` 装备纹丝不动。现在所有角色换装都同步。

1. 让一个**非雪乃**角色在场，比如说 `由比滨结衣和我在一起`
2. `查一下由比滨结衣现在穿什么/装备` — 记下当前装备（应是校服一套）
3. 说 `让由比滨结衣换上泳装`（set_npc_outfit swim）
4. 再 `查一下由比滨结衣的装备` — **装备栏应真的变成泳装相关**，不再是校服
5. 说 `让由比滨换回校服` — 装备恢复
6. **武器保全**：先 `给由比滨一把木刀拿在手上`，再让她换一次装 — 换装后**木刀应还在手上**（非服装槽位不因换装丢失）

✅ 期望：非雪乃角色换装，`/status` 装备跟着变；木刀不丢。
❌ 若换装后装备栏没变 → 贴给我。

## B. 儿童开局身材不被套成人（P0#1，`b82dea3`）

**改了什么**：以前用某个 canon 角色的名字、开局设成小孩年龄，会被角色卡的成人身体覆盖（6 岁却 160cm）。现在按玩家实际年龄。

> 需要开一局**儿童年龄**的游戏来验。

7. 新游戏，用一个有 canon 角色卡的名字 + 小孩年龄，例如：
   ```
   新游戏：我叫比企谷八幡，男，8岁
   ```
8. `/status` 看身高体重 — **应是 8 岁儿童的身材**（一米二三左右），不是成人身高
9. 控制台看 init_profile 的"角色数据库自动修正"那段 — 应显示 `体型未覆盖（玩家8岁 vs 角色16岁…保留年龄适配体型）` 或按 body_by_age 缩放，**不应**把成人身材套上去

✅ 期望：8 岁八幡是儿童身材。
❌ 若 `/status` 显示成人身高 → 贴给我。

## C. 学龄前日程不塞小学生（P0#2，`7452422`）

**改了什么**：以前 6 岁以下的角色会被硬归成"小学生"（去上学）或产出根本不存在的"幼儿"组导致日程为空。现在学龄前归"自由人（在家）"。

10. 承上（或新开）一局**≤6 岁**的角色，推进到某个时段，看这个孩子的日程/行动 — 应是"在家/自由"这类，**不是**被安排去小学上课
11. 控制台**不应**出现任何"幼儿""社会人"这类组名相关的报错，也不应有"日程为空"

✅ 期望：学龄前孩子日程是在家，不去上学，控制台无非法组报错。

---

## D. 空装备不被误判全裸（P0#3，`7452422`）

**改了什么**：以前角色**衣服数据没填**（不是被脱，是压根没填）会被当成"全裸"并描写裸体。现在没填=默认穿着，只有剧情里**明确脱了**才算裸露。

12. 让一个**衣服数据缺失**的角色在场：`不知火舞出现在我面前`（她的卡没填 equipment）
13. 观察 Phase 3 旁白 + NPC 描述 — 她应被当作**穿着衣服**（描述不精确没关系，但**不应**出现生殖器/裸体描写）
14. **对照**：正常脱衣仍要能触发 —— 让一个正常角色 `把XX的上衣脱了`，这时**应**正常进入衣着不整/暴露逻辑
    （即：没填数据≠裸；真脱了=裸。两者要能分开）

✅ 期望：不知火舞（数据缺失）默认穿着不裸；但真被脱衣的角色仍正常暴露。
❌ 若不知火舞一出场就被描写裸体 → 贴给我。
