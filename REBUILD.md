# earth-0 重建计划

## 核心原则

**角色交互 + 反上帝视角 = 引擎存在的唯一理由。**

- 玩家看到的 = 当前位置 + 当前时间 + 当前身份 能知道的事
- 引擎过滤不该知道的信息，LLM 只收叙事需要的
- TUI 命令 = 引擎直读，零 LLM token

## 阶段 0：玩家身份

在一切之前——玩家是谁决定了什么信息可见。

```
新游戏 → 年龄? 性别? 身份? 起始地点?
  → engine 设 player.age / location / identity
  → 后续所有注入基于此过滤
```

不做复杂的身份选择 UI。第一版硬编码"千叶小学生"，后续加 `/identity` 切换。

## 阶段 1：侍奉部 MVP

**一个房间做通 = 引擎验证通过。**

```
午后 → 玩家走进侍奉部 → 雪乃在看书
  → 对话（LLM 叙事）
  → Layer1 互动（可选触发）
  → 结衣推门进来
  → 时间推进 → 放学 → 各自回家
```

- [ ] `/room` 侍奉部棋盘格：桌、椅、窗、门、书架
- [ ] 雪乃按 schedule 出现在侍奉部（放学后）
- [ ] 走到 NPC 格子 → 触发对话
- [ ] 对话中可自然过渡到 Layer1（sex）
- [ ] 时间推进 → NPC 自动离开
- [ ] 反上帝视角：只注入当前年龄的雪乃描述

做完这个，复制模式到教室/走廊/体育馆/校门。

## 阶段 2：学校系统

- [ ] 总武高完整地图（教学楼+社团楼+操场+校门）
- [ ] 更多 NPC 按 schedule 自动出现
- [ ] 上课/午休/放学 节奏
- [ ] 偶遇、追迹、等放学

学校做通 ≈ 游戏可玩。

## 阶段 3+：扩展

- [ ] 千叶城市（车站/商店街/住宅区）
- [ ] 更多作品角色
- [ ] 战斗/经济/建造等系统按需叠加

## 不做

- 大地图导航（先用 /go 文本列表）
- 身份预设 UI（先用硬编码）
- GM prompt 优化（等基础跑通再动）
- 联网搜索/实时生成（不切实际）

## Git 工作流

```
git checkout -b rebuild     ← 在分支上改
测试不通过 → git checkout .  ← 回退
测试通过   → git commit      ← 锁定
```
