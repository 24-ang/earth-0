# earth-0 重建计划

## 核心原则

**角色交互 + 反上帝视角 = 引擎存在的唯一理由。**

- 玩家看到的 = 当前位置 + 当前时间 + 当前身份 能知道的事
- 引擎过滤不该知道的信息，LLM 只收叙事需要的
- TUI 命令 = 引擎直读，零 LLM token

## 阶段 0：玩家身份

在一切之前——玩家是谁决定了什么信息可见。

```
新游戏 → 年龄? 性别? 身份? 起始地点?
  → engine 设 player.age / location / identity
  → 后续所有注入基于此过滤
```

不做复杂的身份选择 UI。第一版硬编码"千叶小学生"，后续加 `/identity` 切换。

## 阶段 1：角色交互

**目标：** 玩家能在房间里走动，碰见 NPC，对话。

- [ ] 简化 engine/state.ts — 只保留 Player + NPC 基础
- [ ] 简化 extension.ts — 只保留 /status /look /room /go
- [ ] /room 棋盘格移动（方向键）
- [ ] 走到 NPC 格子 → 自动触发对话
- [ ] NPC 有 schedule（按时间自动切换位置）
- [ ] 反上帝视角：只注入当前房间 NPC 的当前年龄阶段描述

数据保留：`characters.json` `character_stages.json` `rooms.json`

## 阶段 2：场景

**目标：** 空间有感觉——光线、声音、气味、NPC 在做什么。

- [ ] `/room` 显示环境描述（窗外景色、环境音）
- [ ] 不同时间/天气 → 房间描述变化
- [ ] NPC schedule 驱动房间内自然行为（看书、扫地、睡觉）

## 阶段 3：时间

**目标：** 世界不静止。NPC 有自己的生活。

- [ ] commit_turn 推进时间
- [ ] NPC schedule 按时间自动移动
- [ ] 玩家可以观察/偶遇/追踪 NPC
- [ ] 时间线事件（可选，先不做）

## 阶段 4+：系统叠加

战斗/经济/偷窃/建造/Layer1/IF线……
每加一个系统，当场测试。不通过就回退。

## 不做

- 大地图导航（先用 /go 文本列表）
- 身份预设 UI（先用硬编码）
- GM prompt 优化（等基础跑通再动）
- 联网搜索/实时生成（不切实际）

## Git 工作流

```
git checkout -b rebuild     ← 在分支上改
测试不通过 → git checkout .  ← 回退
测试通过   → git commit      ← 锁定
```
